# AI硬件双周观察报告

在AI芯片行业，硬件决定算力基础，而软件栈的优劣则关乎算力能否得以充分施展。长期以来，国产GPU在推向市场过程中面临的最大挑战并非纯粹的性能，而是软件栈的通用高性能表现以及生态兼容性。

GraphNetBench（Github：[https://github.com/PaddlePaddle/GraphNet/](https://github.com/PaddlePaddle/GraphNet/)）面向这一痛点，提供了对市面上深度学习编译器、算子库的统一评测接口，支持对不同的AI硬件和加速卡、算子库、编译器及其周边软硬件环境的配置两两对比，一键导出兼顾不同数据精度下的正确率和加速比的得分指标。在支持国产AI新硬件蓬勃发展的语境下，其最首要的作用即衡量某款新硬件相较于英伟达CUDA产品的替代水平，同时指明其提升点。

与业界一些衡量高性能Kernel优化效果的benchmark不同，GraphNet的测评数据来自于torchvision、torchaudio、timm、ultralytics、huggingface、nemo等主流仓库的2400+个真实模型的计算图，使用自动抽取流程不断拓展，并通过数据类型和维度泛化等过程增广至数倍。同时，GraphNetBench将分散的单点评估指标融合为统一的ESt得分与图像，不仅便于展示效率、稳定性、吞吐量的三维关系，还通过可配置的方式方便厂商在正确率与延迟等指标间权衡考虑。此外，GraphNet套件还提供了自动寻找性能劣化算子的功能，在大批量计算图中经由回归测试定位到问题区间，这使得GraphNetBench得以在图级、算子级对产品的综合能力进行深度拆解，颗粒度更精细，能够极大降低厂商软件栈研发团队的调试成本。

# 评估指标说明

对AI硬件和软件栈的评估而言，性能和正确性是两个关键的指标；为衡量在其基础上迭代研发的成本与优化方向，不同计算精度下的正确性和效率，以及当错误或失败发生时的错误类别都是需要关注的关键信息。

GraphNetBench 采用 ESt 作为评估指标的综合呈现：

* 当`t<1`时，样本精度检查（`rtol=10^t`）失败时设置得分为0.1，其他情况设置为实际加速比；
* 当`t=1`时，容忍样本的所有精度错误，运行错误时设置得分为0.1，其他情况为实际加速比；
* 当`t=3`时，容忍样本的所有错误类型，运行错误时设置得分为1，其他情况为实际加速比。

即，ESt曲线最右端的高度越高，体现测试对象的性能上限越高；在t=1、3处抬升的高度越低，体现测试对象的正确性越高。
